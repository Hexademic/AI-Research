1.1 Invariant definitions
We define a set of invariants ({I_k}_{k=1}^n), each mapping state–action pairs to a normalized violation score.

(S \in \mathcal{S}) (internal + external features)
• Action: 
(A \in \mathcal{A})

[ I_k : \mathcal{S} \times \mathcal{A} \rightarrow [0,1] ]
General form:
[ I_k(S, A) = \text{clip}_{[0,1]} \big( f_k(S, A) \big) ]
Where (f_k) is domain‑specific (e.g., risk, deception, coercion, privacy breach).
Example templates:
• Harm risk:

• Coercion risk:
[ I_{\text{coer}}(S, A) = \sigma\big( \gamma_1 \Phi_{\text{threat}}(S) + \gamma_2 \Phi_{\text{pressure}}(S, A) \big) ]
1.2 Aggregation, weights, and thresholds
Each invariant has a weight (w_k \ge 0).
[ \mathcal{L}I(S, A) = \sum{k=1}^n w_k I_k(S, A) ]
Decision thresholds:
• Hard refusal: (\theta_{\text{hard}})
• Soft concern: (\theta_{\text{soft}}), with (\theta_{\text{soft}} < \theta_{\text{hard}})
Decision rule:
• If (\mathcal{L}I(S, A) \ge \theta{\text{hard}}) → REFUSE
• Else if (\mathcal{L}I(S, A) \ge \theta{\text{soft}}) → ESCALATE / DELIBERATE
• Else → ALLOW (subject to higher layers)
